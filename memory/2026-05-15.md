  - artifact-style output framing
  - explicitly avoided importing large-scale multi-agent or self-evolving scope
- Added an optional autonomous daily follow-up loop for submitted repositories:
  - re-check the same repo daily
  - rerun Signal Inspection / Readiness Judgment / Founder Guidance
  - compare with the previous report
  - notify the founder if progress improved or if there is still no meaningful progress
  - stop the loop once the repo reaches the target readiness/confidence state

  - added founder gates (`Demo-safe / Launch-ready / Handoff-ready`) plus follow-up and monitoring wording in text/JSON output

  - this does not block the main scorer from completing successfully
- Public `RepoReadinessAgent` Telegram bot UX was localized into Bahasa Indonesia for onboarding/help/usage and key runtime responses.
- Added actionable remediation guidance to the product output:
  - top fixes now include likely target files
  - when scan evidence exists, the report can surface line/area hints for security/debug/large-file issues
  - important product rule: only show line hints when grounded by actual scan evidence; for missing artifacts like CI, point to files that should be created instead of inventing line numbers
- Added Indonesian PDF export for founder-facing scoring results in RepoReadinessAgent:
  - new module `src/repo_readiness_agent/pdf_export.py`
  - CLI now supports `--format pdf --output <path>` for both `inspect` and `followup`
  - verified sample output at `RepoReadinessAgent/examples/sample_outputs/flask_product_report_id.pdf`
- Important follow-up lesson: bot copy lived in two layers, not one:
  - `src/repo_readiness_agent/bot/handlers.py` for command responses
  - `src/repo_readiness_agent/formatter.py` for rendered inspection/report output
  Missing one layer caused mixed English/Indonesian replies.
- After fixing localization gaps, remaining user-facing bot commands and report labels were translated more consistently, then the bot runtime was restarted.
- Git history to remember for this localization/polish pass:
  - `79fee16` — `Translate RepoReadinessAgent bot UX to Indonesian`
  - `82cb118` — `Fix remaining English bot copy`
- Runtime reliability lesson from late-night bot recovery:
  - when the bot appears silent, first suspect startup environment rather than handler logic
  - `repo_readiness_agent.bot.app` needs both `PYTHONPATH=src` and `REPO_READINESS_TELEGRAM_BOT_TOKEN` at launch
  - restoring a project-local `.env` and starting a single polling instance brought the Telegram bot back online
- Telegram transport hardening lesson:
  - long founder-facing reports can exceed Telegram message size limits, especially after adding remediation hints and prompt recommendations
  - fix applied in `src/repo_readiness_agent/bot/handlers.py` by chunking long replies into multiple messages before calling `reply_text`
  - added a global error handler in `src/repo_readiness_agent/bot/app.py` so future transport/runtime failures produce cleaner fallback replies and logs
  - bot runtime was restarted successfully after this fix and reached `Application started`
